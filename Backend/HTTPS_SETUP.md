# 🔐 Настройка HTTPS для работы камеры на мобильных устройствах

## Проблема
Многие мобильные браузеры требуют HTTPS для доступа к камере. Без HTTPS камера не будет работать.

## Решение
Используем самоподписанный SSL-сертификат для локальной сети.

---

## 📋 Пошаговая инструкция

### Шаг 1: Установите зависимости (если нужно)

Если у вас нет OpenSSL, установите библиотеку cryptography:
```powershell
pip install cryptography
```

### Шаг 2: Сгенерируйте SSL-сертификат

```powershell
cd "C:\Users\<ВАШ_ПОЛЬЗОВАТЕЛЬ>\OneDrive\Desktop\Pet-Projects\AgroProjectWeb\Backend"
python generate_cert.py
```

Это создаст папку `certs/` с файлами:
- `cert.pem` - сертификат
- `key.pem` - приватный ключ

### Шаг 3: Запустите Backend с HTTPS

Теперь **не нужно** запускать отдельный фронтенд-сервер: FastAPI отдаёт и сайт, и API.

В терминале:
```powershell
cd "C:\Users\<ВАШ_ПОЛЬЗОВАТЕЛЬ>\OneDrive\Desktop\Pet-Projects\AgroProjectWeb\Backend"
python start_https_server.py
```

Или вручную:
```powershell
uvicorn app.main:app --host 0.0.0.0 --port 8000 --ssl-keyfile "certs/key.pem" --ssl-certfile "certs/cert.pem" --reload
```

### Шаг 4: Откройте сайт на телефоне

В браузере телефона откройте:
```
https://192.168.1.188:8000
```

⚠️ **ВАЖНО**: При первом открытии браузер покажет предупреждение о безопасности, потому что сертификат самоподписанный. Это нормально для локальной сети!

**Что делать:**
- **Chrome/Edge на Android**: Нажмите "Дополнительно" → "Перейти на сайт (небезопасно)"
- **Safari на iOS**: Нажмите "Дополнительно" → "Перейти на сайт"
- **Firefox**: Нажмите "Дополнительно" → "Принять риск и продолжить"

После этого камера должна работать! 🎉

## 🔧 Альтернативные методы

### Метод 1: Использовать OpenSSL (если установлен)

```powershell
# Создать приватный ключ
openssl genrsa -out certs/key.pem 2048

# Создать самоподписанный сертификат
openssl req -new -x509 -key certs/key.pem -out certs/cert.pem -days 365 -subj "/CN=localhost"
```

### Метод 2: Использовать mkcert (рекомендуется для разработки)

1. Установите mkcert: https://github.com/FiloSottile/mkcert
2. Установите локальный CA:
   ```powershell
   mkcert -install
   ```
3. Создайте сертификат:
   ```powershell
   mkcert 192.168.1.188 localhost 127.0.0.1
   ```
4. Переименуйте файлы:
   ```powershell
   move 192.168.1.188+2.pem certs/cert.pem
   move 192.168.1.188+2-key.pem certs/key.pem
   ```

## ❓ Частые проблемы

### Браузер не доверяет сертификату
Это нормально для самоподписанных сертификатов. Просто нажмите "Продолжить" при предупреждении.

### Камера все еще не работает
1. Убедитесь, что сайт открыт через **HTTPS** (не HTTP)
2. Проверьте, что оба сервера (backend и frontend) запущены с HTTPS
3. Попробуйте другой браузер (Chrome обычно работает лучше)

### Ошибка "SSL: CERTIFICATE_VERIFY_FAILED"
Это нормально для самоподписанных сертификатов. Браузер предупредит, но вы можете продолжить.
