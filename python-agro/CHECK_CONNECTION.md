# 🔍 Проверка подключения с телефона

## Проблема: localhost работает, но с телефона не открывается

### ✅ Быстрая проверка

1. **Убедитесь, что сервер запущен с `--host 0.0.0.0`**
   - Должно быть: `uvicorn app.main:app --host 0.0.0.0 --port 8000`
   - НЕ должно быть: `--host 127.0.0.1` или без `--host`

2. **Проверьте IP адрес**
   ```powershell
   ipconfig | findstr IPv4
   ```
   Должен показать `192.168.1.188` (или другой IP в вашей сети)

3. **Проверьте, что телефон и компьютер в одной Wi-Fi сети**

4. **Проверьте файрвол Windows**
   - Откройте "Брандмауэр Защитника Windows"
   - "Разрешить взаимодействие с приложением"
   - Найдите Python и разрешите для "Частные сети"

---

## 🔧 Решение проблем

### Проблема 1: "Не удается подключиться" или таймаут

**Причина:** Файрвол блокирует порт 8000

**Решение:**
1. Откройте PowerShell **от имени администратора**
2. Выполните:
   ```powershell
   netsh advfirewall firewall add rule name="Python Uvicorn" dir=in action=allow protocol=TCP localport=8000
   ```

Или через графический интерфейс:
1. Панель управления → Система и безопасность → Брандмауэр Windows
2. Дополнительные параметры
3. Правила для входящих подключений → Создать правило
4. Тип: Для порта → TCP → Указанные локальные порты: 8000
5. Действие: Разрешить подключение
6. Профили: Частная сеть
7. Имя: "Python Uvicorn"

### Проблема 2: Открывается, но показывает ошибку

**Если запущен с HTTPS, но открываете HTTP:**
- Используйте: `https://192.168.1.188:8000` (не `http://`)

**Если запущен БЕЗ HTTPS, но открываете HTTPS:**
- Используйте: `http://192.168.1.188:8000` (не `https://`)

### Проблема 3: "ERR_CONNECTION_REFUSED"

**Причина:** Сервер не слушает на всех интерфейсах

**Решение:** Убедитесь, что запускаете с `--host 0.0.0.0`:
```powershell
python start_server.py
# или
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

## 🧪 Тестирование

### Шаг 1: Проверьте на компьютере
```powershell
# Должно работать
curl http://192.168.1.188:8000
# или откройте в браузере
# http://192.168.1.188:8000
```

### Шаг 2: Проверьте с телефона
- Откройте браузер на телефоне
- Введите: `http://192.168.1.188:8000`
- Должен открыться сайт

### Шаг 3: Если не работает
1. Проверьте, что оба устройства в одной Wi-Fi сети
2. Попробуйте пинг с телефона (если есть приложение для пинга)
3. Проверьте файрвол Windows

---

## 📱 Альтернативные способы проверки

### Способ 1: Используйте другой порт
Если порт 8000 блокируется, попробуйте 8080:
```powershell
uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
```
И откройте: `http://192.168.1.188:8080`

### Способ 2: Проверьте через другой браузер на телефоне
Иногда проблема в кэше браузера. Попробуйте:
- Chrome
- Firefox
- Safari (iOS)

### Способ 3: Используйте мобильные данные
Если Wi-Fi не работает, попробуйте подключить телефон к мобильной точке доступа компьютера (если поддерживается).

## ✅ Чек-лист

- [ ] Сервер запущен с `--host 0.0.0.0`
- [ ] IP адрес правильный (`192.168.1.188`)
- [ ] Телефон и компьютер в одной Wi-Fi сети
- [ ] Файрвол разрешает порт 8000
- [ ] Используете правильный протокол (http:// или https://)
- [ ] Браузер на телефоне не блокирует подключение

---

## 🆘 Если ничего не помогает

1. Перезапустите сервер
2. Перезагрузите роутер Wi-Fi
3. Попробуйте другой порт (8080, 3000, 5000)
4. Проверьте настройки роутера (AP isolation должен быть выключен)
