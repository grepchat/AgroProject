# 📱 Решение проблемы: не открывается с телефона

## Проблема
`localhost` работает, но `http://192.168.1.188:8000` не открывается с телефона.

## ✅ Решение

### Вариант 1: Автоматическая настройка файрвола (рекомендуется)

1. **Запустите скрипт от имени администратора:**
   ```powershell
   # Правой кнопкой на файле -> "Запуск от имени администратора"
   Desktop\python-agro\fix_firewall.ps1
   ```

2. **Или вручную через PowerShell (от администратора):**
   ```powershell
   New-NetFirewallRule -DisplayName "AgroVision-Port-8000" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
   ```

### Вариант 2: Настройка файрвола вручную

1. Откройте **"Брандмауэр Защитника Windows"**
   - Нажмите `Win + R`
   - Введите `wf.msc` и нажмите Enter

2. Создайте правило:
   - Слева: **"Правила для входящих подключений"**
   - Справа: **"Создать правило..."**
   - Тип правила: **"Для порта"**
   - Протокол: **TCP**
   - Порт: **8000**
   - Действие: **"Разрешить подключение"**
   - Профили: отметьте все (Домен, Частная, Публичная)
   - Имя: **"AgroVision Port 8000"**

3. Нажмите **"Готово"**

### Вариант 3: Временное отключение файрвола (только для теста!)

⚠️ **ВНИМАНИЕ:** Это небезопасно! Используйте только для проверки.

1. Откройте **"Брандмауэр Защитника Windows"**
2. Слева: **"Включение и отключение брандмауэра Защитника Windows"**
3. Временно отключите для частной сети
4. **После проверки обязательно включите обратно!**

## 🔍 Дополнительные проверки

### 1. Проверьте, что сервер запущен с правильным хостом

В терминале должно быть:
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```

Если видите `127.0.0.1:8000` - это неправильно! Нужно перезапустить с `--host 0.0.0.0`

### 2. Проверьте IP адрес

```powershell
ipconfig | findstr /i "IPv4"
```

Должен показать `192.168.1.188` (или другой IP в вашей сети)

### 3. Проверьте, что порт слушается

```powershell
netstat -an | findstr ":8000"
```

Должно быть:
```
TCP    0.0.0.0:8000           0.0.0.0:0              LISTENING
```

Если видите `127.0.0.1:8000` - сервер не принимает внешние подключения!

### 4. Проверьте сеть

- Телефон и компьютер должны быть в **одной Wi-Fi сети**
- Проверьте на телефоне: Настройки → Wi-Fi → название сети должно совпадать

### 5. Попробуйте ping с телефона

На телефоне установите приложение для ping (например, "Network Tools") и проверьте:
```
ping 192.168.1.188
```

Если ping не проходит - проблема в сети, не в файрволе.

## 🚀 После настройки файрвола

1. **Перезапустите сервер** (если был запущен)
2. **Откройте на телефоне:**
   - HTTP: `http://192.168.1.188:8000`
   - HTTPS: `https://192.168.1.188:8000` (если используете HTTPS)

## ❓ Если всё ещё не работает

### Проверка 1: Другой браузер на телефоне
Попробуйте Chrome, Firefox, Safari

### Проверка 2: Другой порт
Попробуйте запустить на другом порту:
```powershell
uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
```
И откройте: `http://192.168.1.188:8080`

### Проверка 3: Проверьте антивирус
Некоторые антивирусы блокируют входящие подключения. Временно отключите для проверки.

### Проверка 4: Роутер блокирует
Некоторые роутеры блокируют подключения между устройствами. Проверьте настройки роутера (AP Isolation / Client Isolation).

---

## 📝 Быстрая команда для проверки

На компьютере откройте PowerShell и выполните:
```powershell
Test-NetConnection -ComputerName 192.168.1.188 -Port 8000
```

Если показывает `TcpTestSucceeded : True` - порт открыт и доступен.
