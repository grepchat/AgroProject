# 🔧 Решение проблем

## Проблема: Ссылка не открывается

### Проверка 1: Сервер запущен?

Убедитесь, что сервер запущен. В терминале должно быть:
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### Проверка 2: Правильный URL?

**На телефоне:**
- HTTPS: `https://192.168.1.188:8000`
- HTTP: `http://192.168.1.188:8000` (если запущен без HTTPS)

**На компьютере:**
- HTTPS: `https://localhost:8000` или `https://127.0.0.1:8000`
- HTTP: `http://localhost:8000` или `http://127.0.0.1:8000`

### Проверка 3: Предупреждение о сертификате?

Если используете HTTPS, браузер покажет предупреждение. **Нужно нажать "Продолжить"** или "Дополнительно" → "Перейти на сайт".

### Проверка 4: Файрвол блокирует?

Windows может блокировать порт. Разрешите Python через файрвол:
1. Откройте "Брандмауэр Защитника Windows"
2. "Разрешить взаимодействие с приложением"
3. Найдите Python и разрешите для частных сетей

### Проверка 5: Проверьте API endpoint

Откройте в браузере:
- `http://192.168.1.188:8000/docs` - должна открыться документация API
- `http://192.168.1.188:8000/api/detect` - должна быть ошибка метода (это нормально)

Если `/docs` открывается, но главная страница нет - проблема в пути к фронтенду.

### Проверка 6: Логи сервера

Посмотрите в терминале, что показывает сервер при запросе. Должны быть строки типа:
```
INFO:     127.0.0.1:xxxxx - "GET / HTTP/1.1" 200 OK
```

Если видите `404 Not Found` - проблема в маршрутах.
Если видите `500 Internal Server Error` - проблема в коде.

## Частые ошибки

### "Connection refused" или "Не удается подключиться"
- Сервер не запущен
- Неправильный IP адрес
- Файрвол блокирует

### "ERR_CERT_AUTHORITY_INVALID" (только HTTPS)
- Это нормально для самоподписанного сертификата
- Нажмите "Продолжить" в браузере

### "404 Not Found"
- Проверьте URL
- Убедитесь, что сервер запущен на правильном порту

### Белый экран или JSON вместо интерфейса
- Фронтенд не найден
- Проверьте путь к `frontend-agro/index.html`
- Посмотрите логи сервера

## Альтернативный способ проверки

Если ничего не помогает, попробуйте запустить без HTTPS:

```powershell
cd Desktop\python-agro
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

И откройте: `http://192.168.1.188:8000`

Это поможет понять, проблема в HTTPS или в чем-то другом.
