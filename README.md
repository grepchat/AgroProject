# AgroVision (AgroProjectWeb) — веб‑система для детекции и подсчёта растений

Этот проект — “сайт + сервер”, который позволяет:
- загружать фото с теплицы или включать камеру,
- автоматически **находить растения на изображении** (нейросеть),
- **считать количество** найденных растений,
- показывать результат в веб‑интерфейсе (рамки вокруг объектов + статистика),
- отдавать данные через API (чтобы можно было подключить другие системы).

Ниже — инструкция, рассчитанная на человека без IT: что здесь за папки, что за что отвечает, и как запустить так, чтобы работало “из коробки”.

---

## Что нужно установить (один раз)

На Windows:
- **Python 3.10+** (лучше 3.10/3.11)
- (желательно) **Git** — чтобы скачать проект (если вы получаете проект архивом, Git не нужен)

Важно: проект использует нейросеть YOLO, поэтому при первом запуске могут устанавливаться “тяжёлые” библиотеки (это нормально).

---

## Самый простой запуск (рекомендуется)

### 1) Откройте терминал в папке `Backend`

Папка проекта у вас примерно такая:
`C:\Users\<ВАШ_ПОЛЬЗОВАТЕЛЬ>\OneDrive\Desktop\Pet-Projects\AgroProjectWeb\`

Дальше команды:

```powershell
cd "C:\Users\<ВАШ_ПОЛЬЗОВАТЕЛЬ>\OneDrive\Desktop\Pet-Projects\AgroProjectWeb\Backend"
```

### 2) Установите зависимости (первый запуск)

```powershell
python -m pip install --upgrade pip
pip install -r app\requirements.txt
```

### 3) Запустите сервер

#### Вариант A (с телефона будет работать камера): HTTPS

Сначала один раз сгенерируйте сертификат:

```powershell
python generate_cert.py
```

Потом запускайте сервер:

```powershell
python start_https_server.py
```

Открывайте:
- на компьютере: `https://localhost:8000`
- на телефоне (в той же Wi‑Fi сети): `https://<IP_компьютера>:8000`

При первом заходе браузер покажет предупреждение о безопасности (самоподписанный сертификат). Это нормально для локальной сети: нужно нажать “Дополнительно” → “Перейти”.

#### Вариант B (для компьютера, без камеры на телефоне): HTTP

```powershell
python start_server.py
```

Открывайте:
- `http://localhost:8000`

## Как узнать IP компьютера (чтобы открыть с телефона)

В PowerShell:

```powershell
ipconfig | findstr /i "IPv4"
```

Обычно IP выглядит как `192.168.x.x`.

## Что вы увидите после запуска

Когда сервер запущен, в браузере по адресу `http(s)://...:8000` откроется веб‑страница **AgroVision**:
- вкладка **“Загрузка изображения”**: выбрали фото → “Запустить анализ” → получите рамки + число растений.
- вкладка **“Живая камера”**: включает камеру и отправляет кадры на сервер (на телефоне часто нужен HTTPS).

Также доступна документация API:
- `http(s)://...:8000/docs` (удобная страница с кнопками “Try it out”)

## Как устроен проект (по модулям)

Проект специально разделён на модули, чтобы было понятно “что за что отвечает”.

### Модуль 1. `Frontend/` — веб‑интерфейс (то, что видит человек)

**Задача:** показать красивый сайт, дать кнопки “выбрать фото / включить камеру”, показать результат.

Главные файлы:
- `Frontend/index.html` — вся страница и логика на JavaScript.

Что делает интерфейс:
- берёт картинку/кадр,
- отправляет на сервер запрос **POST** в `/api/detect`,
- получает JSON‑ответ (список найденных растений),
- рисует рамки поверх изображения/видео,
- показывает статистику (сколько объектов, какие классы/культуры).

### Модуль 2. `Backend/` — сервер (API + выдача сайта)

**Задача:** принять изображение, прогнать через нейросеть, вернуть результат и отдать сайт.

Запуск/служебные скрипты:
- `Backend/start_server.py` — запуск сервера по HTTP (проще, но камера на телефоне может не работать).
- `Backend/start_https_server.py` — запуск по HTTPS (нужен для камеры на мобильных устройствах).
- `Backend/generate_cert.py` — генерация сертификата для HTTPS (самоподписанный, для локальной сети).
- `Backend/fix_firewall.ps1` — помощь с Windows‑файрволом (если телефон не видит порт).

### Модуль 3. `Backend/app/main.py` — “главный вход” сервера

**Что делает:**
- создаёт приложение FastAPI,
- подключает маршруты API `/api/...`,
- **раздаёт фронтенд**: при открытии `/` отдаёт `Frontend/index.html`,
- отдаёт статические файлы фронтенда через `/static` (если нужны).

Если открыть `/` и вместо сайта вы видите JSON — значит сервер не нашёл `Frontend/index.html`.

### Модуль 4. `Backend/app/api.py` — API (точки входа для запросов)

Главный endpoint:
- `POST /api/detect` — отправляете картинку (JPG/PNG), получаете результат детекции.

Внутри происходит:
- проверка типа файла,
- декодирование изображения (OpenCV),
- выбор модели из реестра,
- запуск детекции,
- возврат результата по строгой схеме (Pydantic‑модели).

### Модуль 5. `Backend/app/models.py` — формат ответа (контракт данных)

**Задача:** описать, в каком виде сервер отдаёт результат.

В ответе есть:
- `total_plants` — сколько объектов нашли,
- `plants[]` — список найденных растений (рамка, уверенность, координаты),
- `meta` — информация о модели и изображении.

Это важно для интеграции: если завтра делать мобильное приложение/1С/дашборд — они будут читать эти поля.

### Модуль 6. `Backend/app/detector/` — “подключение нейронки”

Это слой, который “прячет” детали ML и даёт простой интерфейс: **дать картинку → получить список объектов**.

Файлы:
- `Backend/app/detector/base.py` — базовый интерфейс детектора (как должен выглядеть любой детектор).
- `Backend/app/detector/yolov8.py` — конкретная реализация на YOLOv8 (Ultralytics).
- `Backend/app/detector/registry.py` — реестр моделей: какую модель использовать по культуре (сейчас используется модель “по умолчанию”).

Где лежат веса модели:
- `Backend/weights/yolov8_flowers.pt` — веса (файл нейросети), которые используются по умолчанию.

Если вы добавите новые веса под другую культуру — их обычно кладут в `Backend/weights/` и подключают в `registry.py`.

### Модуль 7. `Backend/app/geometry.py` — перевод координат “из пикселей в метры”

Сейчас там простая формула (пример-заглушка):
- берём центр рамки в пикселях,
- переводим в “метры теплицы” линейно.

В реальном проекте сюда обычно добавляют калибровку камеры/матрицу преобразования.

### Модуль 8. `TrainingAndDatasets/` — подготовка данных и оценка качества (дообучение)

Эта папка нужна не для работы сайта, а для работы с датасетами/качеством модели:
- `TrainingAndDatasets/prepare_flowers_yolo.py` — превращает исходные изображения в структуру YOLO‑датасета (images/labels + YAML).
- `TrainingAndDatasets/evaluate_counts.py` — скрипт оценки качества по “подсчёту” (MAE/MAPE) на валидации.
- `TrainingAndDatasets/clean_flowers_yolo.py` — короткий тест, видит ли Python вашу видеокарту (CUDA).

**Важно:** дообучение YOLO обычно делается отдельной командой Ultralytics (не в этом сервере). Логика дообучения не встроена в API, чтобы сервер оставался простым и стабильным.

## “Частые проблемы” (если у человека не запускается)

### 1) Открывается `/docs`, но главная страница показывает JSON

Это значит: сервер работает, но **не найден фронтенд**.
Проверьте, что в проекте есть файл:
- `Frontend/index.html`

### 2) С телефона не открывается `http(s)://<IP>:8000`

Обычно причины:
- телефон и компьютер **не в одной Wi‑Fi сети**
- Windows‑файрвол блокирует порт 8000

Решение: см. `Backend/FIX_MOBILE_ACCESS.md`.

### 3) Камера на телефоне не включается

На телефоне часто нужно:
- открывать сайт по **HTTPS**
- принять предупреждение сертификата

Решение: запускайте `Backend/start_https_server.py` (и один раз `generate_cert.py`).

## Для разработчика (кратко)

API:
- `POST /api/detect` принимает `multipart/form-data` с полем `image`.

- FastAPI отдаёт `Frontend/index.html` по `/`,
- API доступно по `/api/...`.
